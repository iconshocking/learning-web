PNPM Useful Commands

  • init
  • add
  • update: version numbers in pnpm config work as such: ^ up to the next major version, ~ means up to the next minor version, and no symbol means the exact version
  • audit: check for vulnerabilities
  • ls [dependency]: describe dependencies and any transitive dependencies (add --depth option to see transitive deps); call just ls to run the command on the project
  • run [script-name]: (can exclude script name to list all scripts instead)

Package Manager Scripts

Define any script in scripts inside package.json to be able to invoke it via pnpm run [script-name]
