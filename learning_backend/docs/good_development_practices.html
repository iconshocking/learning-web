Good development practices

Behavior-Driven Development (BDD) vs Test-Driven Development (TDD)

BDD is from the user's POV and describes expectations and desires. TDD is from the developer's POV and focuses on implementation and functionality.

BDD should be used to delineate the happy path. Note unhappy paths as the branches not taken (to make sure unknowns are accounted for).

Rapidly prototype the happy path to uncover 'unknown unkowns', which can be added to unhappy path enumeration.

TDD should then be used to fully flesh out the happy path, followed by the unhappy paths by severity.
